# FPGA-Based Real-Time Audio Equalizer

Project for Reconfigurable Computing.

Team: Daniel Gonzales, Ethan Florendo, Isaak Truong

## Overview

A real-time 3-band audio equalizer implemented on the PYNQ-Z2 FPGA board. Audio is captured through the onboard ADAU1761 codec, processed in the frequency domain via FFT/IFFT on the programmable logic, and output in real time. Gain for the low, mid, and high frequency bands is adjustable at runtime through a Python/Jupyter interface.

## Hardware

- **Board:** PYNQ-Z2 (Zynq XC7Z020)
- **Audio Codec:** ADAU1761 (3.5 mm line-in / line-out)
- **Interface:** I2S (BCLK = 3.072 MHz, LRCLK = 48 kHz)
- **Headset:** Runmus K8

## Signal Processing Pipeline

```
Line-In → ADC → I2S RX → [FFT → Band Gain → IFFT] → I2S TX → DAC → Line-Out
                              (Programmable Logic)
```

## Build Instructions

This repository contains the Vivado project files for an FPGA-based audio equalizer design targeting the PYNQ-Z2 / Zynq-7020 platform.

The project includes the Vivado project file and source directory needed to reopen the design:

```text
FFT.xpr
FFT.srcs/
```
Generated Vivado folders such as .runs, .cache, .gen, .hw, .sim, and .ip_user_files are intentionally not included because they are large and can be regenerated by Vivado.
```
Requirements
Xilinx Vivado installed
PYNQ-Z2 board files installed in Vivado
Target board: PYNQ-Z2 / Zynq-7020
```
If the PYNQ-Z2 board does not appear in Vivado, install the PYNQ-Z2 board files before opening the project.

Opening the Project
Clone or download this repository.
Open Vivado.
Select:
File → Project → Open
Open the project file:
FFT.xpr
Vivado should load the project sources from:
FFT.srcs/
Regenerating Output Products

Because generated folders are not included, Vivado may show some IP or block-design output products as out of date.

After opening the project:

Open the Block Design.
Click Validate Design.
If prompted, regenerate output products.
Generate the HDL wrapper if needed:
Right-click block design → Create HDL Wrapper
Run synthesis:
Flow Navigator → Run Synthesis
Run implementation:
Flow Navigator → Run Implementation
Generate bitstream:
Flow Navigator → Generate Bitstream
Exporting for PYNQ

After bitstream generation, copy the generated .bit and .hwh files to the PYNQ board.

The files are usually located under a path similar to:

FFT.runs/impl_1/

You will need both:
```
design_name.bit
design_name.hwh
```
Rename them so they have the same base name, for example:
```
audio_equalizer.bit
audio_equalizer.hwh
```
Then copy them to the PYNQ board, for example:

/home/xilinx/jupyter_notebooks/audio_equalizer/
Important Notes

This repository does not include generated Vivado build folders:
```
FFT.runs/
FFT.cache/
FFT.gen/
FFT.hw/
FFT.sim/
FFT.ip_user_files/
```
These folders are created automatically by Vivado when synthesis, implementation, simulation, or bitstream generation is run.

If Vivado reports missing generated files, regenerate the output products from inside Vivado.

Main project components include:

I2S receiver and transmitter blocks
ADAU1761 audio codec interface
AXI GPIO gain control
Audio sample extraction and repacking RTL
FFT/IFFT processing chain
AXI-stream helper modules
Known Development Status

The current design validates the major digital audio-streaming components:

I2S transmitter accepts valid AXI audio words
I2S receiver decodes known serial test data
RX-to-TX AXI-stream bypass is functional
Audio sample extractor and repacker are functional
Gain control GPIO is present

The full FFT/IFFT streaming path is still under debug. Current testing shows valid audio framing at the output, but sample payloads may become zero after the FFT chain. The next debugging step is to probe each FFT-chain stage individually.

Rebuilding the Project

To rebuild from a clean checkout:

Open FFT.xpr in Vivado.
Let Vivado update/regenerate IP output products.
Validate the block design.
Regenerate HDL wrapper if needed.
Run synthesis.
Run implementation.
Generate bitstream.
Export/copy .bit and .hwh to the PYNQ board.
